Construye algoritmos para la solución de problemas

Un algoritmo es un conjunto de instrucciones paso a paso que permiten resolver un problema. Se pueden representar con pseudocódigo o diagramas de flujo.

  • Identificación del problema
  • Diseño del algoritmo
  • Representación gráfica o textual
  • Pruebas y depuración

Etapas para construir un algoritmo:

  • Entemder el problema: Definir claramente qué se quiere resolver. Analizar entradas y salidas: ¿Qué datos necesita? ¿Qué se espera obtener?
  • Diseñar el algoritmo:
  • Diagrama de flujo: Representa gráficamente el proceso paso a paso usando símbolos.
  • Pseudocódigo: Representa el algoritmo en forma textual, similar al lenguaje humano.
  • Probar el algoritmo: Antes de programar, pruébalo con valores reales para ver si funciona correctamente.
  • Optimizarlo: Después de que funcione, puedes:

    -Simplificar operaciones.

    -Usar variables eficientes.

    -Eliminar pasos innecesarios.

Aplica estructuras de control con un lenguaje de programación

Las estructuras de control permiten tomar decisiones y repetir instrucciones en un programa. Los más comunes son:

  • Condicionales (if, else, switch): Un condicional simple es una estructura de control que
    ejecuta un conjunto de líneas de código si es cierta una expresión booleana.
    A un condicional simple se le puede añadir la cláusula else para especificar qué lineas de código se quieren ejecutar si la expresión booleana es falsa.

  • Bucles (for, while): For
    Cuando se sabe o se puede saber el número de repeticiones While
    Cuando no se sabe el número de repeticiones Do While
    Cuando no se sabe el número de repeticiones, pero se debe realizar al menos una vez

  • Control de flujo: Las estructuras de control de flujo son conjuntos de instrucciones que determinan el orden de ejecución de un programa.
    Permiten que el código avance secuencialmente, tome decisiones o repita acciones.

Aplica estructuras de datos con un lenguaje de programación

Las estructuras de datos son formas organizadas de almacenar y gestionar la información para que pueda ser utilizada de forma eficiente dentro de un programa.
Aplicarlas correctamente permite que el software sea más ágil, ordenado y fácil de mantener.

  • Arreglos o listas: Permiten almacenar varios datos del mismo tipo bajo un mismo nombre, y se accede a cada uno mediante un índice.
    Se usan, por ejemplo, para manejar listas de números, nombres o calificaciones.

  • Pilas y colas
  • Diccionarios o mapas: Son estructuras más avanzadas. La pila funciona bajo el principio de “el último en entrar, primero en salir”,
    y la cola funciona como “el primero en entrar, primero en salir”. Se usan para controlar secuencias, como procesos en espera o historial de acciones.

  • Listas enlazadas: Son colecciones ordenadas de elementos que pueden crecer o disminuir en tamaño. A diferencia de los arreglos, las listas permiten agregar o quitar elementos fácilmente.
